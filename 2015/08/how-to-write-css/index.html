CSS 设计模式

OOCSS

OOCSS（Object-Oriented CSS，面向对象的CSS），将页面的UI组件看作对象，再以这些对象为单位考虑CSS的可重用性。

OOCSS的目的主要是减少对 HTML 结构的依赖，增加 CSS class 重复性的使用。

SMACSS

在 SMACSS 中，将CSS的样式归为以下5类：

  • 基础（Basic）
  • 布局（Layout）
  • 模块（Module）
  • 状态（State）
  • 主题（Theme）

基础规则

基础规则定义了网站的基本央视，比如重置样式、元素级别的通用样式（如超链接的颜色）等，是定义网站整体默认样式的规则。

布局规则

布局规则定义了页面中主要组件（如页面头部、侧边栏、主题内容）的样式。

在专门用于布局的选择器前加上 l- 这样的前缀，可以对选择器的用途加以明示。

模块规则

模块是比布局元素粒度更小的组件，比如导航栏、走马灯组件、对话框等。模块的特征各自独立，其样式与组件在页面中的位置无关。

对于模块，推荐使用 class 选择器而不使用元素选择器。

状态规则

定义组件状态的规则。如标签页的激活和非激活状态，在标签页面切换是，使用形如 .is-active 或 .is-tab-active 这样带有 is- 前缀的 CSS 类定义其样式。

状态规则与模块规则的区别在于，状态样式是在模块样式的默认样式上加以覆盖的样式，而且可以利用JavaScript的点击事件等机制来进行样式的切换。

主题规则

针对网站中某一组页面指定其他配色方案或外观整个换掉是，可以使用主题规则。

加上 theme- 前缀，可以与其他样式类区分开。

BEM

BEM 即 Block、Element、Modifier 的缩写，是一种 class 的命名技巧。

将 Block 区块作为起始开头前缀；Element 则是 Block 下的元素；Modifier 则是这个元素的属性。

不同 Block 和 Element 用 __ 两个底线区隔开来，不同的 Modifier 则用 -- 两个 dash 区隔。至于 - 一个 dash 则表示这个 class 不依赖任何 Block 或 Element，是个独立的存在，例如：.page-container 或 .article-wrapper。
