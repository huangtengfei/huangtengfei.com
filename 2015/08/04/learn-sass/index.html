Sass 初探

发表于 2015-08-04   |  

安装与使用

安装

Sass 需要 Ruby 运行环境，在 Windows 上使用需要先安装 Ruby。

然后，在命令行下输入命令：

gem install sass

使用

Sass 命令的格式如下：

sass [选项] [输入文件] [输出文件]

也可以让 Sass 监听某个文件或目录，一旦源文件有变动，就自动生成编译后的版本：

// 监控文件
sass --watch input.scss:output.css
// 监控目录
sass --watch dir/sass:dir/css

Sass 提供四个编译风格的选项：

  • nested：嵌套缩进的 CSS 代码，它是默认值。
  • expanded：没有缩进的、扩展的 CSS 代码。
  • compact：简洁格式的 CSS 代码。
  • compressed：压缩后的 CSS 代码。

Expanded 格式是可读性最好的，开发环境中可使用。生产环境当中，一般使用最后一个选项。

sass --style compressed test.sass test.css

运算

Sass支持四则运算、关系运算符（<、>、 <=、 >=）及等值判断（==、！=）。

需要注意的是，进行除法运算时，要将表达式作为变量抽出，或者用括号括起来，不然编译器会原样输出。

注释

标准的多行CSS注释 /* comment */ ，会保留到编译后的文件。但压缩模式（:compressed）会被删除。

单行注释 // comment，只保留在SASS源文件中，编译后被省略。

使用 /*! comment */，表示这是”重要注释”。即使是压缩模式编译，也会保留这行注释，通常可以用于声明版权信息。

混合（@mixin + @include）

混合指令可以用来定义可重用的代码块。声明时使用 @mixin ， 调用时使用 @include [混合名] 。混合的声明部分不会出现在编译后的 CSS 文件中。

混合的强大之处，在于可以指定参数和缺省值。

继承（@extend）

Sass 允许一个选择器，继承另一个选择器。

编译后，继承和被继承的选择器会呈现并列关系，使用同一个属性块。继承能将多个重复的选择器归纳到一起，相比混合而言，代码可以少一些。

如果编译后不想生成被继承的选择器样式，可以使用选择器占位符（%），针对抽血的选择器进行继承。

导入（@import）

@import 指令能够将指定的文件合并到样式表中，编译成单个CSS文件。

被导入的Sass文件同时也会各自被单独编译成独立的CSS文件，如果不需要这样的独立CSS文件，可以加一个下划线作为文件名的前缀，这样Sass会将其作为片段来处理。

在导入时，不需要加下划线前缀和文件扩展名。

注意事项

避免过度使用嵌套

由于CSS选择器是从右向左解析的，因此在定义样式时，嵌套的深度不宜超过3层。如果超过3层，需要对模块的粒度重新进行推敲。

选择器的定义可参考 【写出优雅的CSS】 一文

避免过度使用混合

对于那些不是用参数，编译后生成的代码固定不变的混合，应考虑使用多重样式类的方案代替。尽力不使用不带参数的混合。

避免过度使用继承

过度使用继承会产生大量的选择器，比如上文继承中的例子，如果有很多不同颜色的按钮，将对每个颜色的按钮生成一个样式。此时，使用 <div class="button button-yellow"> 这样的多重样式类的扩展性更好。
