高效组织CSS代码

组织CSS代码文件

将通用类和业务类的代码放置于不同目录，通常css下包含lib文件夹、业务css文件夹和default.css、common.css、ie-style.css

+css
 ++lib
 ++business
 --default.css
 --common.css
 --ie-style.css

default.css的作用是重置元素的默认样式，目的是让元素在各浏览器中有统一的外观。

common.css中放置共通模块的样式和基础样式。共通模块包括页面对话框、提示框等共通组件样式以及页面头部、底部、侧边栏等共通UI样式。基础样式包括全局的页面布局设置、字体设置、背景和前景色等，以及一些用以提高代码复用度的公有class（如设置字体不可见的样式）。

ie-style.css中放置兼容旧版IE浏览器的样式。以减轻非IE浏览器加载样式表的负担，并方便在不需要兼容时移除。在页面中使用IE浏览器独有的注释引用此样式文件：

其他CSS样式文件可按照具体业务放置不同文件夹，并进来保证每个模块对应的样式文件不超过200行，如果过大可继续划分为更小的模块（如business.header.css、business.content.css、business.footer.css）。这样划分是为了方便开发，在发布时可压缩为一个文件。

组织CSS代码

样式声明的顺序应按照模块中元素的层级关系来处理，同一级的元素按照页面位置从上到下、从左到右定义。先声明共通样式，然后声明个体样式。

另外，可以使用动态样式语言Less和Sass编写高复用性和高扩展性的CSS代码。

使用CSS Reset：统一浏览器显示效果

前面default.css的作用就是CSS Reset，在重置样式时主要考虑以下几个方面：

1）HTML5中的新标签

因为在IE8/IE9中没定义HTML5中新标签的默认display样式，因此需要重置这些标签的display样式。

2）padding、margin和border

标签在浏览器之间的差异主要是由与这三个样式有关的默认样式产生，因此也才会有使用

这个流行的重置方案。

3）字体设置
<h1>~<h6>、<strong>、<em>等语义化的标签都有默认字体，然后在实际项目中，一般都会重定义这些样式。

4）其他元素的样式重置

比如列表项、表格元素和链接的重置等。

关于CSS Reset可参考《Web前端开发最佳实践》6.2章节示范。

给CSS样式定义排序

通常有三种方式，按类型分组排序、按字母顺序排序和按定义长度排序，推荐使用第一种。

按类型顺序排序是指把CSS属性分成显示与浮动、定位、尺寸、边框相关属性、字体样式、背景和其他样式7大类，把影响元素布局的样式（如float、margin、padding、height、width等）排在前面，把不影响布局的样式（如background、color、font等）放到后面，主次分明。

使用CSScomb工具可以对CSS排序。

合理利用CSS的权重：提高代码重用性

CSS样式的基础选择器有6种：

  • ID选择器（如#reader_title{}）
  • 类选择器（如.reader_title{}）
  • 属性选择器（如a[href=’http://www.w3.org']）
  • 伪类和伪对象选择器（如:hover{} ::after{}）
  • 标签类型选择器（如a{}）
  • 通配选择器（如body * {}）

所有在CSS样式中定义的选择符都是由这6种基础选择符组合而成的，组合方式有三种：

  • 后代选择符（如.reader .title{}）
  • 子选择符（如.reader>.title{}）
  • 相邻选择符（如.reader+.title{}）

基础选择器具有这样的优先级（前者是后者10倍）：
ID ＞ 类|伪类|属性选择 > 标签类型|伪对象 > 通配符

定义选择符的原则是尽量使选择符的权重低，目的是保证样式在应用于多个元素时容易被覆盖。具体原则有以下三个：

1）尽量不要使用ID选择器
2）减少子选择器的层级
3）使用组合的CSS类选择器

兼容IE浏览器

1）熟悉IE浏览器中常见的样式兼容问题

参见 9种在IE中常见的兼容问题

2）分离样式兼容代码

方法一：使用条件注释，让不同浏览器加载不同的样式文件以达到让兼容代码和正常代码分离的目的，参见组织CSS代码小节的示例。

方法二：为提高兼容样式的优先级，可以在根元素（<html> 或者 <body>）上针对不同浏览器添加不同的样式类。

em、px还是%

px为绝对单位，表示一个像素点。em和%为相对单位，em等同于元素上计算的字体大小值，%是相对于父元素尺寸的大小来计算的。

在设置元素尺寸和字体大小时，有以下几个原则：

1）尽量设置相对尺寸

设置行高一般使用em，而设置高度或者宽度则一般使用百分比。

2）只有在可预知元素尺寸的情况下才使用绝对尺寸

比如设计上要求网页整体宽度固定、侧边栏宽度固定、页头页尾高度固定等，或者在展示图片、视频时要求固定尺寸。

3）使用em设置字体大小

在继承层级很多的时候，每个层级都设置有相对字体大小会导致不方便预知子元素的实际字体大小，因此，可使用rem单位（IE8不支持），rem是相对于根元素的字体大小来计算的，可以避免以上问题。

高效的CSS

使用高效的CSS选择器

在一个选择符中，最右边的选择器是关键选择器，关键选择器决定着浏览器初始匹配的元素数量。由于CSS选择器的匹配顺序是从右向左，因此定义CSS时最好遵循以下原则：

  • 避免使用通配符（*）
  • 避免使用标签选择器及单个属性选择器作为关键选择器
  • 不要在ID选择器前使用标签名（这是多余的）
  • 不要在选择符中定义过多的层级（最好不超过三层）

CSS相关的图片处理

1）不要设置图片的尺寸

一是超过实际尺寸的图片在网络传输中会占用更多带宽，二是在图片缩放时占用额外的CPU计算，所以最好的做法是针对原始图片单独做一套缩略图。

2）使用CSS Sprite技术

CSS Sprite通过合并图片能避免多次的图片请求，但存在开发和维护繁琐（调整图片位置）、使用不当导致性能的问题（合并后的图片过大时），因此需要注意：

  • 在项目后期使用CSS Sprite技术
  • 合理组织Sprite图（把相关的背景图放到同一张Sprite图中）
  • 控制Sprite图的尺寸和大小（长乘宽不超过2500，尺寸不超过200）
  • 合理控制背景图但也之间的距离以及背景图的位置（防止当背景图尺寸比实际应用的元素尺寸小时，背景图中出现其他无关的背景图）
  • 借助工具（使用Sprite Cow）

减少CSS代码量

除了压缩代码以为，在开发过程中注意精简代码。

1）合并相关CSS规则

比如字体样式 font-family、font-style、font-size可以合并为一个 font 样式。

2）定义简洁的属性值

比如颜色值#FF33EE简化为#F3E、尺寸值0.8em可简化为.8em，尺寸值为0时可省略单位。

3）合并相同的定义

把相同的CSS合并定义，并且利用CSS的继承属性，把一些样式设置在父元素上。

4）删除无效的定义

通过Chrome自带的开发工具可以找到unused CSS rules

其他高性能CSS实践

  • 避免使用@import（使用@import导入新样式文件会阻止页面的并行下载）
  • 避免使用IE独有的样式，如图片滤镜和CSS表达式
