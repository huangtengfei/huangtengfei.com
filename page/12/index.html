一个普通网站的访问过程

参见知乎上Web基本概念介绍中张秋怡同学的 回答 。

  1. 用户操作浏览器访问，浏览器向服务器发出一个 HTTP 请求；
  2. 服务器接收到 HTTP 请求，Web Server 进行相应的初步处理，使用服务器脚本生成页面；
  3. 服务器脚本（利用Web Framework）调用本地和客户端传来的数据，生成页面；
  4. Web Server 将生成的页面作为 HTTP 响应的 body，根据不同的处理结果生成 HTTP header，发回给客户端；
  5. 客户端（浏览器）接收到 HTTP 响应，通常第一个请求得到的 HTTP 响应的 body 里是 HTML 代码，于是对 HTML 代码开始解析；
  6. 解析过程中遇到引用的服务器上的资源（额外的 CSS、JS代码，图片、音视频，附件等），再向 Web Server 发送请求，Web Server 找到对应的文件，发送回来；
  7. 浏览器解析 HTML 包含的内容，用得到的 CSS 代码进行外观上的进一步渲染，JS 代码也可能会对外观进行一定的处理；
  8. 用户与页面交互（点击，悬停等等）时，JS 代码对此作出一定的反应，添加特效与动画；
  9. 交互的过程中可能需要向服务器索取或提交额外的数据（局部的刷新，类似微博的新消息通知），一般不是跳转就是通过 JS 代码（响应某个动作或者定时）向 Web Server 发送请求，Web Server
    再用服务器脚本进行处理（生成资源or写入数据之类的），把资源返回给客户端，客户端用得到的资源来实现动态效果或其他改变。

BFC 及 浮动

发表于 2015-08-05   |  

BFC（块级格式化上下文）

创建了 BFC的元素就是一个独立的盒子，里面的子元素不会在布局上影响外面的元素，反之亦然，同时BFC任然属于文档中的普通流。

BFC的应用

不和浮动元素重叠

如果一个浮动元素后面跟着一个非浮动的元素，那么就会产生一个覆盖的现象。而根据 CSS Floats 中的描述，创建了 BFC 的元素不能与浮动元素重叠。问题解决。

清除元素内部浮动

根据 CSS2.1 规范的高度计算规则，在进行普通流中的块级非替换元素的高度计算时，浮动子元素不参与计算。而在计算生成了 BFC 的元素的高度时，其浮动子元素应该参与计算。

所以，只要把父元素设为BFC就可以清理子元素的浮动，使浮动元素闭合。

解决上下相邻两个元素重叠

根据 CSS 规范，生成 BFC 的元素不会和在流中的子元素发生空白边折叠。所以解决这种问题的办法是要为两个容器添加具有BFC的包裹容器。

Compass 初探

发表于 2015-08-04   |  

Sass 本身只是一个编译器，Compass 在它的基础上，封装了一系列有用的模块和模板，补充 Sass 的功能。它们之间的关系类似于 Javascript 和 jQuery。

安装

在已有Ruby的环境下，在命令行输入命令：

gem install compass

使用

compass create

使用 compass create 命令可创建一个 Compass 项目：

compass create myproject

该命令会生成以下目录结构：

  • sass目录，存放 sass 源文件
  • stylesheets目录，存放编译好的 CSS
  • config.rb文件，Compass 的配置文件

发表于 2015-08-04   |  

CSS 设计模式

OOCSS

OOCSS（Object-Oriented CSS，面向对象的CSS），将页面的UI组件看作对象，再以这些对象为单位考虑CSS的可重用性。

OOCSS的目的主要是减少对 HTML 结构的依赖，增加 CSS class 重复性的使用。

结构与外观分离

结构对象，是指那些设定高度和宽度的样式属性。外观是指文字颜色、背景色等样式。

Sass 初探

发表于 2015-08-04   |  

安装与使用

安装

Sass 需要 Ruby 运行环境，在 Windows 上使用需要先安装 Ruby。

然后，在命令行下输入命令：

gem install sass

使用

Sass 命令的格式如下：

sass [选项] [输入文件] [输出文件]

也可以让 Sass 监听某个文件或目录，一旦源文件有变动，就自动生成编译后的版本：

// 监控文件
sass --watch input.scss:output.css
// 监控目录
sass --watch dir/sass:dir/css
